一个专为加密货币交易者设计的实时交易监控与策略模拟终端。融合了暗黑科技美学与专业级风险计算功能，支持多交易所、多主题、OCR识别等高级特性。

## ✨ 核心特性

### 📊 实时监控
- **双交易所支持**：无缝切换 Binance / OKX 实时行情
- **70+ 主流币种**：覆盖 ETH、BTC、SOL、Meme币、Layer2、DeFi 等全品类
- **WebSocket 实时更新**：毫秒级价格推送

### 🛡️ 风险管理系统
- **实时盈亏计算**：多空仓位独立与合并计算
- **动态风险敞口**：净Delta值监控
- **智能强平预警**：可视化风险进度条与风险等级
- **净值监控**：实时权益计算与颜色预警

### 🎮 交互功能
- **三主题切换**：暗黑/明亮/像素三种视觉模式
- **OCR图像识别**：自动从截图识别仓位数据（均价/数量）
- **Telegram集成**：一键发送交易简报与配置推送
- **战术模拟器**：实时模拟加仓后的均价与强平价变化

### 📈 策略工具
- **智能策略矩阵**：自动生成支撑/阻力位（3级阻力+3级支撑）

- **杠杆动态调整**：实时计算杠杆倍数影响

二、基础操作流程

步骤1：选择数据源与币种

1. 点击顶部左侧的 BINANCE 或 OKX 标签选择交易所
2. 从右上角下拉菜单选择交易币种（支持70+种）
3. 观察实时价格更新（需要网络连接）

步骤2：设置仓位参数

1. 输入钱包余额：在"钱包余额(U)"输入框中输入您的USDT余额（默认2800）
2. 设置杠杆倍数：在"杠杆倍数(X)"输入框中设置杠杆（默认52X）
3. 输入多单仓位：
   · 多单均价（例：3775.26）
   · 多单数量（例：3.5 ETH）
4. 输入空单仓位：
   · 空单均价（例：2888.62）
   · 空单数量（例：1.6 ETH）

步骤3：监控核心指标

· 未结盈亏 (PNL)：实时显示浮动盈亏（绿赚红亏）
· 实时净值 (EQUITY)：当前账户总价值（低于500变红）
· 风险敞口 (DELTA)：净风险暴露方向与数值
· 风险进度条：直观显示距离强平价的相对位置
· 风险等级：LOW/HIGH/CRITICAL 三档评估

三、高级功能详解

1. OCR图像识别仓位

1. 点击底部 📷 识别 按钮
2. 选择包含仓位信息的截图
3. 系统自动识别"价格"、"数量"等关键词
4. 自动填充到对应仓位输入框

支持格式：

· 中英文混合识别
· 数字格式（支持小数）
· 常见截图格式（PNG, JPG, WebP）

2. 主题切换

点击左上角的主题按钮（☀️/🌙/👾）循环切换：

· ☀️ 暗黑主题：专业交易终端，适合夜间使用
· 🌙 明亮主题：简约现代，适合白天使用
· 👾 像素主题：复古游戏风格，独特视觉体验

3. 战术模拟器

功能：模拟加仓/减仓后的仓位变化

使用方法：

1. 在"模拟价格"输入目标价格（例：4000）
2. 在"数量"输入模拟数量：
   · 正数：模拟加多仓（例：0.2）
   · 负数：模拟加空仓（例：-0.2）
3. 自动计算结果显示：
   · New AVG：新均价（多/空分别计算）
   · New LIQ：新强平价（基于新仓位重新计算）

4. 策略矩阵自动生成

系统根据当前价格自动计算：

· 3级阻力位：价格 × (1.01, 1.025, 1.05)
· 3级支撑位：价格 × (0.99, 0.975, 0.95)

操作：

· 点击任一策略行 → 自动加载到战术模拟器
· 阻力位：加载负数量（模拟加空）
· 支撑位：加载正数量（模拟加多）

5. Telegram集成配置

1. 点击底部 ⚙️ TG 按钮打开配置面板
2. 获取并输入：
   · Bot Token：通过 @BotFather 创建机器人获取
   · Chat ID：个人/群组的聊天ID
3. 开启 CORS Proxy 推荐（避免跨域问题）
4. 点击 Test Send 测试连接
5. 配置自动保存到浏览器本地存储

快速发送简报：
点击底部 📡 简报 按钮，自动发送包含：

· 当前币种
· 实时价格
· 当前盈亏
· 净值
· Delta值

四、数据解读指南

1. 颜色编码系统

· 🔵 蓝色 (var(--pri))：多头、主色调、正常状态
· 🔴 红色 (var(--dan))：空头、亏损、危险状态
· 🟢 绿色 (var(--safe))：盈利、安全、支撑位
· 🟡 黄色 (var(--warn))：警告、高风险、阻力位
· ⚪ 灰色 (var(--dim))：次要信息、标签、单位

2. 风险指标解读

指标 正常范围 警告范围 危险范围 含义
PNL > 0 (绿) - < 0 (红) 浮动盈亏
EQUITY > 500 (绿) - < 500 (红) 账户净值
风险条 绿色 ≤ 50% 黄色 50-80% 红色 > 80% 距强平距离
RISK LOW HIGH CRITICAL 风险评估

3. Delta值解读

· 正Delta (蓝色)：净多头，希望价格上涨
· 负Delta (红色)：净空头，希望价格下跌
· 零Delta (灰色)：市场中性，价格波动影响小

五、常见使用场景

场景1：日内交易监控

1. 设置当前仓位
2. 切换到明亮主题（白天使用）
3. 开启 Binance 数据源
4. 使用策略矩阵快速查看关键位
5. 底部简报发送给Telegram群组

场景2：仓位调整模拟

1. 看到价格接近支撑位
2. 从策略矩阵点击对应支撑位
3. 战术模拟器自动加载参数
4. 调整模拟数量查看新均价和强平价
5. 决定是否实际加仓

场景3：风险管理检查

1. 查看风险进度条颜色
2. 确认风险等级为 LOW
3. 检查净值是否大于500
4. 监控Delta值是否符合预期方向
5. 如风险过高，考虑减仓

场景4：多平台对比

1. 点击 🔄 换源 快速切换交易所
2. 对比不同交易所价格差异
3. 如价差明显，可能存在套利机会
4. 注意不同交易所的强平规则可能不同

六、最佳实践

1. 首次使用设置

1. 配置Telegram机器人（便于发送警报）
2. 根据使用环境选择合适的主题
3. 确认WebSocket连接正常（价格实时更新）
4. 测试OCR功能确保截图识别准确

2. 日常使用流程

1. 打开页面，检查实时价格更新
2. 更新仓位数据（或使用OCR导入）
3. 查看核心监控面板的风险状态
4. 使用策略矩阵规划下一步操作
5. 如有需要，使用战术模拟器测试加仓方案

3. 风险管理建议

· 净值警戒线：设置500U为最低警戒线
· 风险条警戒：当进度条超过50%（变黄）时关注
· Delta控制：保持Delta在可承受范围内
· 定期简报：每小时发送一次简报到Telegram

4. 性能优化

· 网络不佳时：可暂时关闭CORS Proxy
· 大量仓位时：先使用战术模拟器测试，再实际操作
· 多币种监控：建议为每个主要交易币种保存浏览器书签
